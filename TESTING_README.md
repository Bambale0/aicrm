## 🧪 Типы тестов

### 1. Unit тесты
- Тестируют отдельные функции и методы
- Используют моки для внешних зависимостей
- Быстрые и изолированные

### 2. Интеграционные тесты
- Тестируют взаимодействие между сервисами
- Проверяют полный рабочий процесс
- Более медленные, но более реалистичные

### 3. Тесты производительности
- Проверяют время отклика
- Тестируют нагрузку
- Мониторят использование ресурсов

## 🛠️ Инструменты тестирования

### pytest
- Основной фреймворк для тестирования
- Поддержка async/await
- Плагины для покрытия и отчетов

### pytest-asyncio
- Поддержка асинхронных тестов
- Автоматическое обнаружение async функций

### pytest-cov
- Анализ покрытия кода
- HTML и XML отчеты
- Проверка минимального покрытия

### unittest.mock
- Мокирование внешних зависимостей
- AsyncMock для асинхронных функций
- MagicMock для сложных объектов

## 🔧 Конфигурация тестирования

### pytest.ini
```ini
[tool:pytest]
testpaths = src/aicrm/tests
python_files = test_*.py
python_classes = Test*
python_functions = test_*
addopts =
    --verbose
    --tb=short
    --strict-markers
    --asyncio-mode=auto
markers =
    integration: marks tests as integration tests
    slow: marks tests as slow
    unit: marks tests as unit tests
```

### Конфигурация покрытия
```ini
[coverage:run]
source = src/aicrm
omit =
    */tests/*
    */venv/*
    */__pycache__/*

[coverage:report]
exclude_lines =
    pragma: no cover
    def __repr__
    raise AssertionError
    raise NotImplementedError
```

## 📋 Лучшие практики

### Написание тестов
1. **Один тест - одна ответственность**
2. **Использовать descriptive имена**
3. **Тестировать happy path и edge cases**
4. **Изолировать тесты (моки, фикстуры)**
5. **Проверять ожидаемое поведение**

### Структура тестов
```python
class TestServiceName:
    @pytest.fixture
    def service_fixture(self):
        # Настройка сервиса для тестов
        pass

    def test_method_name_success(self, service_fixture):
        # Тест успешного сценария
        pass

    def test_method_name_failure(self, service_fixture):
        # Тест сценария неудачи
        pass

    def test_method_name_edge_case(self, service_fixture):
        # Тест граничного случая
        pass
```

### Мокирование
```python
@pytest.fixture
def mock_external_service(self):
    with patch('module.path.Service') as mock:
        mock.return_value = Mock()
        mock.method = AsyncMock(return_value=expected_result)
        yield mock
```
