# Стадии автоматизации (Stages)

## Обзор
Страница управления стадиями автоматизации позволяет настраивать этапы бизнес-процессов с помощью drag & drop интерфейса.

## Функциональность

### 🎯 Просмотр стадий
- **Визуальное отображение** стадий в виде блоков
- **Drag & drop** для изменения порядка
- **Цветовые индикаторы** для разных типов стадий
- **Группировка по процессам**

### ➕ Создание стадии
- **Выбор типа сущности** (order, customer, task)
- **Настройка имени и описания**
- **Выбор цвета** для визуального отличия
- **Привязка к процессу автоматизации**

### ✏️ Редактирование стадии
- **Изменение параметров** стадии
- **Переназначение цвета**
- **Изменение порядка** в процессе
- **Обновление описания**

### 🗑️ Удаление стадии
- **Проверка зависимостей** (активные роботы, триггеры)
- **Безопасное удаление** с подтверждением

### 🔗 Привязка к процессам
- **Связь со стадиями** других процессов
- **Условные переходы**
- **Параллельные ветки** выполнения

## API Endpoints

```typescript
GET    /automation/stages/           // Список стадий
POST   /automation/stages/           // Создание стадии
GET    /automation/stages/{id}       // Получение стадии
PUT    /automation/stages/{id}       // Обновление стадии
DELETE /automation/stages/{id}       // Удаление стадии
```

### Перетаскивание стадий
```typescript
// Изменение порядка стадий в процессе
onDragEnd: (result) => {
  const { source, destination } = result;
  // Пересчет order_index для всех стадий

```

### Валидация перемещений
- **Проверка зависимостей** между стадиями
- **Валидация бизнес-логики**
- **Предотвращение циклов**

## Типы стадий

### 🔄 Автоматические стадии
- **Выполняются автоматически** при переходе
- **Интеграция с внешними сервисами**
- **Обработка ошибок**

### 👤 Ручные стадии
- **Требуют подтверждения** от пользователя
- **Уведомления ответственным**
- **Временные ограничения**

### 🤖 AI-стадии
- **Использование ИИ** для обработки
- **Анализ контента**
- **Автоматическое принятие решений**

## UX/UI особенности

- **Drag & drop анимации** для плавности
- **Визуальные связи** между стадиями
- **Предпросмотр изменений**
- **Откат операций** при ошибках

## Интеграция с другими компонентами

### 🤖 Роботы (Robots)
```typescript
// Стадии определяют контекст для роботов
stage_id: number  // Привязка робота к стадии
```

### 🎯 Триггеры (Triggers)
```typescript
// Триггеры срабатывают при переходе на стадию
target_stage_id: number  // Целевая стадия триггера
```

### 📊 Логи автоматизации
```typescript
// Каждая стадия имеет логи выполнения
stage_name: string  // Имя стадии в логах
```
